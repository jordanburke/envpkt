---
title: envpkt init
description: Generate an envpkt.toml template in the current directory
---

Generate a new `envpkt.toml` file with optional identity section, catalog reference, or scaffolding from fnox.

## Usage

```bash
envpkt init [options]
```

## Options

| Option                  | Description                                                              |
| ----------------------- | ------------------------------------------------------------------------ |
| `--from-fnox [path]`    | Scaffold `[secret.*]` entries from `fnox.toml` (optionally specify path) |
| `--catalog <path>`      | Path to shared secret catalog                                            |
| `--identity`            | Include `[identity]` section                                             |
| `--name <name>`         | Name (requires `--identity`)                                             |
| `--capabilities <caps>` | Comma-separated capabilities (requires `--identity`)                     |
| `--expires <date>`      | Credential expiration `YYYY-MM-DD` (requires `--identity`)               |
| `--force`               | Overwrite existing `envpkt.toml`                                         |

## Examples

```bash
# Basic template
envpkt init

# With identity section
envpkt init --identity --name "my-agent"

# Full identity with capabilities
envpkt init --identity --name "bot" --capabilities "read,write" --expires "2027-01-01"

# Reference a shared catalog
envpkt init --catalog "../infra/envpkt.toml"

# Scaffold from fnox.toml
envpkt init --from-fnox
```
